Aroma matrix

This page is two reference grids built from your extracted kitchen books (not live NotebookLM). The spice matrix lists herbs and spices in rows; each column is one of eight aroma families (G1–G8). A gold ● means the Aroma extract assigns that spice to that family; a faint · means it is not counted there. That is different from “this spice goes with chicken”: it is about how the spice smells, grouped for the sheet.

After cross-book data loads, you also get a Harmony number (how many other spices are said to harmonize with this one in the pairing matrix) and Books A·F·T (whether the same ingredient appears in the Aroma file, the Flavor Bible merge, or the Flavor Thesaurus wheel). Tap any spice row (not only the name link) to open a drawer with pairings, harmonizes-with, and more. Switch to Harmony heatmap to see, per aroma column, how many of those partners sit in each group (numbers and tint), instead of ●/·.

The food matrix below is separate: rows are foods (chicken, rice, tomatoes…), columns are spices. A ● means that extract lists the spice as a seasoning for that food. Use Priority 25 for a compact sheet; All spices for the full index. See the boxed legends under each toolbar for the full picture.

How to read the spice matrix

Read across a row for one spice. The first column is its name (the link goes to the full Aroma profile). The next eight columns are not foods and not “how strong” the spice is — they are aroma families from the reference classification. Think of them as columns on a printed aroma chart.

  • G1–G8 (● and ·) — Labels under each header (Grassy, Citrus, …) come from the extract and aroma_data/aroma_matrix_meta.json. ● = this spice is tagged with that aroma group in ingredients.json. · = not tagged for that group (empty does not mean “bad”, only “not in that bucket” for this sheet).
  • Harmony (# partners) — Appears after data loads. One number for the whole row: how many other spices appear as harmony partners in aroma_data/pairing_matrix.json. It answers “how connected is this spice in the spice–spice network?”, not “how many foods”.
  • Books (A·F·T) — Three letters show coverage in other extracts, not quality. A lit = row exists in the Aroma ingredient list. F lit = same id has a Flavor Bible (or merged) entry in combined_data/ingredients_unified.json. T lit = name appears on the Flavor Thesaurus wheel. A dim letter means that source has no row for this id (common for obscure spices).
  • Harmony heatmap mode — Same eight columns, different meaning: each cell is how many of this spice’s harmony partners belong to that aroma group. Darker / larger numbers = more partners in that family. Switch back to Aroma groups for the original ●/· view.
  • Row tap — Opens an inline panel with harmonizes-with links, Flavor Bible tiers (if any), Thesaurus family, and a short “bundled library context” blurb when present in the unified file.
More detail: files and limits
  • Data is offline JSON in this repo (GitHub Pages). It reflects whatever last ran through the extract + merge scripts, not your live NotebookLM session.
  • The Flavor Thesaurus pairing graph (thousands of links) is not drawn as a matrix here; only wheel family + unified row show up in the drawer and T.
  • Science of Cooking, SFAH, and Art of Flavor / Fermentation snippets live inside ingredients_unified.json as kitchen_context and in the drawer’s “Bundled library context” section, not as extra matrix columns.

Column wording is aligned with a Vierich/Vilgis-style aroma sheet (e.g. aroma_matrix_sheet_1.pdf); G numbers match ingredients.json aroma_groups and Aroma lookup badges. The drawer pulls from combined_data/ingredients_unified.json (schema v2: ingredients rows + optional kitchen_context bundle). This UI is a working grid, not a pixel-perfect PDF copy.

Food × spice pairings

This grid answers a different question than the spice matrix above: “For this food, which spices does the Aroma extract list as seasonings?” Each row is one food (or dish category); each column is one spice or herb. Data comes from aroma_data/food_pairings.json. Priority 25 limits columns to the same curated spice set as the sheet; All spices uses every indexed spice (very wide on small screens — scroll horizontally).

Spice columns

Priority 25 = spice columns match the sheet’s priority list. All spices = one column per spice in ingredients.json (can be 100+ columns).

How to read the food × spice grid

Scan along a row to see which spices the extract associates with that food. Scan down a column to see which foods use that spice as a listed seasoning. This is still extract data, not a universal truth about cooking.

  • ● = that spice appears in the food’s seasonings list in the JSON.
  • · = not listed for that food in this extract (other books may still recommend it).
  • Headers — First line is shortened if the name is long; second line shows the full spice name when needed. Hover (or screen reader) gives the full title.
  • Tap a food row — Opens a list of all seasonings as chips with links to Aroma profiles.
